# Автономера — збір opendata через BAS (Browser Automation Studio)

Покрокова інструкція: автоматично обходити opendata.hsc.gov.ua з ротацією IP/фінгерпринтів і
делікатно (кожен регіон ~раз на 10 хв) заливати номери в нашу базу. Дані відкриті (ГСЦ), збираємо легально.

## 0. Що робить макрос
BAS відкриває реальний Chromium (проходить Akamai), а наш JS-скрипт (`collect_all.js`) один раз
проходить усі регіони: підставляє регіон → «Весь регіон» → усі типи → забирає ПОВНИЙ список і шле на
наш сервер `/collect-html`. Сервер сам парсить, визначає тип по серії (Додаток 5), прибирає дублі й
оновлює саме ці регіони (і появу, і зникнення). Пауза ~22 c між регіонами → цикл ~10 хв.

## 1. Встановлення BAS
1. Завантаж BAS: https://bablosoft.com/shop/BrowserAutomationStudio (безкоштовна версія підходить).
2. Встанови і запусти. Створи новий проєкт (New project).

## 2. Базовий сценарій (1 потік, 1 проксі — найпростіше)
У редакторі перетягни дії в такому порядку:
1. **Load page** → URL: `https://opendata.hsc.gov.ua/check-leisure-license-plates/`
2. **Wait** 5–8 c (щоб пройшов Akamai; за потреби закрий модалку «Залишитись на основному сайті»).
3. **Execute JavaScript** → встав ВЕСЬ вміст файлу `collect_all.js`. Постав «Wait for result» (чекати
   завершення) і збережи результат у змінну (це лог: скільки залито по кожному регіону).
4. **Log** → виведи цю змінну (побачиш «Вінницька: +12 / -3 …»).
5. **Wait** ~5 c.
6. Обгорни кроки 1–5 у **Loop** (нескінченний). Кожен прохід ≈ 10 хв → кожен регіон оновлюється раз на ~10 хв.

Запуск: кнопка ▶️. Все — далі само.

## 3. Проксі + фінгерпринти (щоб IP не блокувалась)
- BAS → налаштування потоку → **Proxy**: встав список резидентних проксі (по 1 на потік), формат
  `host:port:user:pass`. BAS сам ротує.
- **Фінгерпринт**: увімкни FingerprintSwitcher (BAS-плагін) — він підмінює User-Agent, canvas/WebGL,
  мову, таймзону під кожен потік. У безкоштовній версії базово теж працює.

## 4. Багато IP одночасно (опційно, якщо хочеш швидше / надійніше)
- Постав **2–3 потоки** (Thread count = 2–3), кожному — своя проксі/фінгерпринт.
- ⚠️ Не більше: це держ-сервіс, нам не треба його класти. 1–3 потоки + паузи — достатньо.

## 5. Перевірка
Після першого циклу глянь у Telegram-боті: «📊 Статистика» / «📰 Нові» — мають оновитись цифри.
Або в адмінці. Якщо лог показує `+N / -M` по регіонах — все працює.

## 6. Окремий скрипт на КОЖЕН регіон (`collect_one.js`)
Якщо хочеш кожному регіону — свій потік/проксі (а не один цикл по всіх):
- Відкрий `collect_one.js`, угорі впиши `var REGION = "Вінницька";` (назва) або значення опції (напр. `"2"`),
  або лиши `""` і обери регіон кроком BAS «Select» перед дією.
- Зроби стільки копій дії/потоків, скільки регіонів — кожному свій REGION і своя проксі.
- Один прогін = один регіон (швидко). Зручно ділити навантаження по проксі.

## 7. Чекер проксі (`check_proxy.js`)
Перевіряє, які з твоїх проксі реально проходять Akamai і віддають список (не всі годяться).
BAS-флоу:
1. Заведи список проксі (файл/змінна) і **Loop** по ньому.
2. Усередині циклу: **Set proxy** (поточний) → **Load page** opendata → **Wait** 5–8 c →
   **Execute JavaScript** = вміст `check_proxy.js` (чекати результат у змінну).
3. **If** результат починається з `OK` → **Append to file** `good_proxies.txt` ← цей проксі.
   Інакше — пропускаємо.
Повертає `OK:<рядків>` (проксі робочий) або `FAIL:…`. Далі у зборі (п.2–6) використовуй лише good_proxies.txt.
